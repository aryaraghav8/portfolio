Case Study 01

Internal Web
Portal Redesign

How I improved performance, usability, and consistency across 3 internal dashboards used by mixed cross-functional teams at Accuster Technologies.

CompanyAccuster Technologies
RoleUI/UX & Frontend Dev
TimelineNov 2023 – Oct 2025
PlatformWeb — React + Figma
Figma React UI/UX Performance Design Systems Usability Testing
01 — The Problem

What was going wrong?

The internal portals used daily by cross-functional teams — operations, finance, and product — were suffering from critical performance bottlenecks. Pages took too long to load, causing frustration and lost productivity. Beyond speed, there was no shared design language, forcing users to re-learn interactions every time they switched portals.

🐢
Slow Load Times

Unoptimized assets, bloated components, and no lazy loading — directly impacting team productivity every day.

🧩
No Reusable Components

Each portal built in isolation with duplicated code, making updates expensive and error-prone.

🎭
Inconsistent UI Patterns

Different navigation patterns and layouts across portals caused confusion and a steep learning curve.

📉
Poor Dev Handoff

No structured design documentation meant developers freely interpreted specs, leading to mismatched builds.

02 — The Process

How I approached it

A structured end-to-end process — starting from user discovery, through system-level design, and ending with measurable performance improvements.

01
Discovery & Audit

Audited all 3 portals — mapped user flows, identified pain points, and benchmarked load times using Chrome DevTools and Lighthouse scores.

02
Wireframing & IA

Created 15 wireframes in Figma to restructure navigation and layouts, making key actions accessible within fewer clicks for all team types.

03
Design System

Built a shared component library — buttons, tables, forms, cards — with consistent spacing, typography, and color tokens across all portals.

04
Frontend Build

Rebuilt in React with modular components. Implemented code-splitting, lazy loading, and asset optimization to directly attack load time.

05
Usability Testing

Ran sessions with users from each department, iterated on navigation and layouts, and validated a 20% task success improvement.

03 — The Solution

What I built

A cohesive, performant portal suite with shared design language, modular React architecture, and structured handoff documentation.

⚡
Performance Optimized Frontend

React code-splitting, lazy loading, and image compression — reducing average page load time by 30% across all portals.

🧱
Shared Component Library

Reusable UI components in Figma and React — visual consistency across portals, reducing future development effort significantly.

🗂️
Redesigned Dashboards

3 clean, scalable dashboards with clearer information hierarchy, better data visualisation, and role-aware layouts for mixed teams.

📋
Structured Dev Handoff

Annotated Figma specs, component docs, and interaction notes — reducing back-and-forth between design and dev teams.

04 — The Impact

Results that mattered

Measurable impact on both performance and experience, validating the combined design + development approach.

30%
Reduction in frontend load time across all 3 portals
20%
Improvement in task success rate after usability testing
15+
Wireframes & prototypes delivered for rapid validation
05 — Learnings

What I took away

Performance is a design problem too — slow interfaces erode trust just as fast as confusing UI. Looping engineers in early surfaces constraints before they become blockers.

Building a design system before touching any portal paid dividends immediately. Consistency became effortless once a shared visual language existed.

Mixed-team users have wildly different mental models. Testing with real users from each department surfaced issues no design review alone would have caught.
