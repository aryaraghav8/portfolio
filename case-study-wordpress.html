Case Study 02

WordPress
UI Enhancements

How I fixed inconsistent styling across 4 live WordPress websites to create a unified, professional brand experience that users could trust.

CompanyAccuster Technologies
RoleUI/UX Designer & Frontend Dev
Scope4 WordPress Websites
FocusConsistency & Engagement
WordPress UI/UX CSS Figma Responsiveness Navigation
01 — The Problem

A brand with no consistent voice

Four live WordPress websites — each built at different times by different people — had grown into a fragmented collection of mismatched styles. Users jumping between them encountered completely different visual languages, making the brand feel unprofessional and unreliable.

The inconsistency wasn't just cosmetic. Navigation patterns differed across sites, font sizes were arbitrary, spacing felt random, and button styles changed from page to page — directly hurting user trust and engagement.

Before
  • Different font families used across pages
  • Inconsistent button colors and sizes
  • Mismatched spacing and layout grids
  • Navigation menus varied per site
  • Brand colors applied inconsistently
  • No visual hierarchy — everything felt equal
After
  • Unified typography system across all sites
  • Consistent button styles and CTA patterns
  • Standardized spacing and layout rules
  • Cohesive navigation structure applied
  • Brand color palette enforced globally
  • Clear visual hierarchy guiding user attention
02 — The Scope

4 sites, one system

Each website had unique content needs, but all needed to speak the same visual language. Here's what was specifically addressed per site:

01
Site One

Rebuilt navigation menu structure and standardised header/footer components for a more predictable user flow.

02
Site Two

Unified typography — replaced mixed font usage with a clean type scale, improving readability across all devices.

03
Site Three

Fixed spacing inconsistencies and applied a consistent grid system — pages felt structured and breathable.

04
Site Four

Redesigned CTA buttons and form elements to match the brand palette, increasing visual clarity and click intent.

03 — The Process

How I tackled it

01
Visual Audit Across All 4 Sites

Screenshotted every major page and laid them side-by-side. Catalogued every inconsistency — fonts, colors, spacing, buttons — into a shared audit document.

02
Defined a Shared Style Guide

Created a lightweight Figma style guide covering typography scale, color tokens, spacing rules, and component patterns — the single source of truth for all 4 sites.

03
Implemented via Custom CSS

Applied changes through WordPress theme customisation and custom CSS — adjusting global styles first, then page-level overrides without breaking existing layouts.

04
Navigation & Responsiveness Fixes

Standardised mobile nav patterns, fixed hamburger menu inconsistencies, and ensured all sites rendered correctly across all screen sizes.

05
Cross-browser QA & Review

Tested each site across Chrome, Firefox, Safari, and Edge. Validated visual consistency and interaction states before final sign-off.

04 — Impact

Consistency that converts

Bringing 4 sites under one visual system made the brand feel credible, coherent, and trustworthy to every visitor.

05 — Learnings

What I took away

A style guide is worth creating even for small projects. Defining tokens and rules upfront pays back immediately when applying changes across multiple sites simultaneously.

Users notice inconsistency even when they can't name it. The feeling of "something being off" directly affects how much they trust a brand — consistency is a trust signal.

WordPress's flexibility is a double-edged sword. Without governance it becomes a source of entropy. Working within platform constraints while enforcing design discipline is a real, learnable skill.
